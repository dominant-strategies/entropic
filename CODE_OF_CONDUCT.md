# Code of Conduct

## Our Standard

Be direct, respectful, and technically honest.

Expected behavior includes:

- discussing ideas and code without personal attacks
- making review feedback specific and actionable
- assuming good faith while still holding a high technical bar
- helping contributors understand decisions and constraints

Unacceptable behavior includes:

- harassment, intimidation, or discrimination
- doxxing or sharing private information without consent
- abusive, insulting, or bad-faith review behavior
- repeated disruption of project discussions

## Scope

This applies to project spaces, including issues, pull requests, discussions,
and other official communication channels.

## Enforcement

Project maintainers may remove, edit, or reject contributions or participation
that violate this code of conduct.

## Reporting

For conduct issues, contact the maintainers privately instead of escalating in
public issue threads.
